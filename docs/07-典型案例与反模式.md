# 07 · 典型案例与反模式

> **这一篇是血泪教训集**。
> 来自 wxcbrc 项目反向阅读的真实发现 + 业界知名反模式。
> 每条都是一次"踩坑→发现→修复"循环。

---

## 1. wxcbrc 案例摘要（按 12 篇资产）

下表把 wxcbrc 项目的 12 篇资产**关键发现**压缩成可教学的卡片。

### 01-系统总览

| 维度 | 数字 |
|---|---|
| 代码量 | 22 个 Model / 19 个 Mapper / 15 个 Controller / 51 个 Vue 组件 |
| 技术栈 | Java 8 + Spring Boot + Vue 2 + Element UI + 静态 jQuery |
| 子项目 | 3 个独立（后端 / 后台管理 / 静态前台） |
| 数据库 | MySQL 18 表（命名不统一：业务表用下划线，业务老表无下划线） |
| 关键问题 | 端口冲突（bootstrap 9090 vs application 9091） |

**教训**：技术栈版本落后（Java 1.8 / Vue 2.6）→ 越早识别越主动。

### 02-数据模型

| 发现 | 教训 |
|---|---|
| `BaseModel` 是 Long id，但 `NewsInfo.news_id` 是 String | **id 类型必须统一** |
| `linksinfo` / `passipinfo` / `imenu` / `iimenu` 表名无下划线 | **命名规范从一开始就定** |
| 业务字段 `links_ip` 实际存 url | **字段名错位是定时炸弹** |
| `yk_liulan` 拼音字段 | **业务字段必须英语** |
| `MenuInfo` Model 没人用 | **无用 Model 必删** |

**金句**：数据库命名是**最难改**的事，越早治理越省力。

### 03-API 清单

| 发现 | 教训 |
|---|---|
| 4 处 `@Api(tags="用户管理接口")` 错标 | **复制粘贴陷阱，必用枚举或常量** |
| 公开端点 `@CrossOrigin` 7 处未做白名单 | **@CrossOrigin 应配 allowedOriginsPatterns** |
| 45 个权限串但 4 处错用 | **权限串应在枚举中定义** |
| 端点约 70 个 | **API 清单是联调事实表** |

**关键发现**：**Swagger tags 错位**是最常见的复制粘贴 bug。

### 04-Mapper 清单

| 发现 | 教训 |
|---|---|
| `NewsInfoMapper.xml#listAll` 用 `${news_title}` | **SQL 注入 P0** |
| `MyBatisBaseDao` 通用方法无 XML 实现 | **抽象基类必须有默认实现或默认实现警告** |
| `SysRoleDeptMapper` 继承基类无任何自定义 | **空继承 = 死代码** |
| `LinksInfo.xml` / `PassIpInfo.xml` 无 Mapper 后缀 | **命名一致性** |
| 46 个 `// TODO Auto-generated method stub` | **IDE 自动生成未清理** |

**关键发现**：**SQL 注入多在"快速查询"代码中**（绕过 ORM 的 SQL 字符串拼接）。

### 05-服务与业务逻辑

| 发现 | 教训 |
|---|---|
| 改密时重置 salt → 改密后立即无法登录 | **改密流程涉及 token 失效** |
| 菜单树递归中 `parentName` 取值错 | **递归 + parent 引用要小心** |
| `saveRpt` 方法未在接口声明 | **接口与实现必须 100% 对应** |
| 标题查重更新时未做 | **CRUD 的 C/U/D 行为必须一致** |

**关键发现**：**改密是高频踩坑点**（salt/token/会话失效/原密码校验）。

### 06-安全认证

| 发现 | 教训 |
|---|---|
| JWT 密钥硬编码 `"hardcoded-default-secret"` | **密钥必外置 + 启动校验非默认** |
| CORS `allowedOrigins("*")` + `allowCredentials(true)` | **CORS 2.4+ 启动失败** |
| `/actuator/**` 全部暴露（含 /env /heapdump） | **actuator 必按需放行** |
| JWT 过期 24h 无刷新 | **JWT 必有刷新机制** |
| `SysLogAspect` Pointcut 写错包名 | **AOP 切面必须有"已触发"测试** |
| `findUserByPassword` 走明文密码 | **不能存/传明文密码** |

**关键发现**：**AOP 切面是最容易被遗忘的"半成品"**——代码在，但永远不触发。

### 07-前端页面与组件清单

| 发现 | 教训 |
|---|---|
| `Login.vue` 用 `res.msg === '交易成功'` 判成功 | **前后端协议要严格**（`res.code === 200`） |
| 搜索按钮 perms 错写 | **perms 应在 enum 中定义** |
| `Passipinfomgmt.vue` 整套 perms 错用 | **复制粘贴要复审** |
| 4 个死代码组件（About/Notice/Message/ThemePicker） | **死代码定期清** |

**关键发现**：**前端权限错用多是复制粘贴**——必用集中常量。

### 08-前端状态与路由

| 发现 | 教训 |
|---|---|
| axios request 拦截器对所有请求校验 token → 登录死循环 | **拦截器必加白名单** |
| token header 用 `token` 而非 `Authorization` | **HTTP header 必走标准** |
| `setPerms` 历史上可能在 menu.js | **store 模块边界要清晰** |
| axios 每次 new instance | **HTTP 客户端必单例** |
| `en_us.json` 缺 6 键 | **多语言必 CI 校验完整性** |

**关键发现**：**axios 拦截器是前端 bug 重灾区**。

### 09-静态前台

| 发现 | 教训 |
|---|---|
| `baseUrl` 写死 `10.52.9.110:9091` | **配置必外置** |
| `[后管登录]` 链接 9090 → 9091 端口错 | **跨子项目配置必同步** |
| `wxcbrc-info.htm` 富文本 `.html()` 无 XSS 过滤 | **前端 XSS 防御** |
| DOCTYPE 显式 IE6 兼容 | **历史包袱要主动卸** |

**关键发现**：**静态站 baseUrl 硬编码是部署期必踩**。

### 10-业务流图

| 发现 | 教训 |
|---|---|
| 新闻发布后未累加 `NewsInfoCount` | **业务期望 vs 实现 必对齐** |
| 新闻详情自动累加 click_count | **业务副作用必在文档中标注** |
| 前台改密 → 命中 IP 白名单 → 拒绝改密 | **白名单语义要清晰（"允许"还是"拒绝"）** |
| 登录用 `res.msg` 判成功 | **业务流文档要明确响应码** |

**关键发现**：**业务流 = 跨层串联**，是新人理解项目的最快路径。

### 11-技术债清单

| 分类 | 数量 | 严重度分布 |
|---|---|---|
| 注释代码块 | 5+ 处 | P3 |
| 命名不一致 | 8+ 处 | P2-P3 |
| 死代码 | 13 处 | P3 |
| TODO 占位 | 46+ 个 | P3 |
| 硬编码 | 6+ 处 | P0-P2 |
| Swagger 错标 | 4 处 | P2 |
| 权限错用 | 5 处 | P1-P2 |
| 性能问题 | 5 处 | P2 |
| 版本落后 | 9 类 | P3 |

**金句**：**技术债数量 > 20 才算"摸到本质"**。

## 2. 业界反模式（高频踩坑）

### 2.1 安全类

| 反模式 | 业界案例 | 教训 |
|---|---|---|
| 密钥硬编码 | Equifax 2017（Apache Struts 未打补丁） | 密钥**必外置 + 启动校验** |
| SQL 注入 | TalkTalk 2015（156 万用户数据泄露） | ORM 是默认，绕过必评审 |
| XSS | 各类 XSS 蠕虫 | 富文本渲染**后端白名单 + 前端 escape** |
| CSRF | 多起银行事件 | 关键操作必 token |
| 权限提升 | 多起垂直越权事件 | **默认拒绝** > 默认允许 |

### 2.2 架构类

| 反模式 | 业界教训 |
|---|---|
| 单体巨石 | Knight Capital 2012 部署事故（45 分钟亏 4.65 亿）——配置不一致 |
| 双主键数据库 | 多起数据不一致事件——**单一事实源** |
| 跨服务分布式事务 | 多起对账事故——**Saga / 幂等 / 补偿** 必有 |
| 微服务过度拆分 | 多起"为拆而拆"反例——**先单体后拆分** |
| 缓存击穿 | 多起雪崩事件——**熔断 + 限流 + 降级** |

### 2.3 工程类

| 反模式 | 教训 |
|---|---|
| 散弹枪修改（Shotgun Surgery） | 一个改动涉及 10+ 文件——**关注点未分离** |
| 依恋情结（Feature Envy） | 一个函数只用别的类的字段——**职责错位** |
| 过长函数 | 100+ 行函数——**必拆** |
| 重复代码 | 同样 if 出现 3+ 次——**抽象** |
| 死代码 | 不被调用的函数——**必删** |
| 魔法数 | `if (status === 3)` 而不命名——**必用枚举/常量** |
| 注释代码块 | 5+ 行 `//` 注释代码——**Git 历史会记得** |
| 长参数列表 | 函数 5+ 参数——**用对象** |

### 2.4 流程类

| 反模式 | 教训 |
|---|---|
| 文档与代码漂移 | 改代码不改文档——**资产 = 代码伴生** |
| 一次性大重构 | 大爆炸——**绞杀者模式 / 增量** |
| 无 owner 模块 | 谁也不敢动——**每模块必 owner** |
| 无 runbook | 故障时手忙脚乱——**SRE 文档必备** |
| 无回滚预案 | 上线后回不去——**蓝绿 / 灰度 / 开关** |

### 2.5 AI 编程类

| 反模式 | 教训 |
|---|---|
| 喂整个仓库给 AI | 上下文超限——**喂本方法论 12 篇资产** |
| 用 AI 写不熟悉的领域 | AI 会自信地胡说——**人必校验** |
| AI 写 P0 修复 | 缺乏风险意识——**P0 必人写** |
| AI 不接资产 | AI 不知道项目规范——**资产是 AI 的肌肉记忆** |
| 资产 = 注释式 | 不带元信息、不带严重度——**资产=结构化** |

## 3. 反复出现的 5 个"老朋友"问题

以下是 5 个**任何老系统都有**的典型问题。

### 老朋友 1：密钥 / 密码硬编码

**症状**：

```java
// JwtTokenUtils.java
String secret = "hardcoded-default-secret";  // ⚠️ P0
```

**修复**：

```java
@Value("${jwt.secret}")
private String secret;
```

```yaml
# application.properties
jwt.secret=${JWT_SECRET:PLEASE_OVERRIDE}
```

```java
@PostConstruct
public void checkSecret() {
    if (secret.length() < 32 || secret.contains("hardcoded-default-secret")) {
        throw new IllegalStateException("jwt.secret is default or too short");
    }
}
```

### 老朋友 2：SQL 字符串拼接

**症状**：

```xml
<select id="listAll">
    SELECT * FROM news_info WHERE news_title = '${news_title}'
</select>
```

**修复**：

```xml
<select id="listAll">
    SELECT * FROM news_info
    <where>
        <if test="news_title != null">
            AND news_title = #{news_title}
        </if>
    </where>
</select>
```

### 老朋友 3：复制粘贴 perms

**症状**：

```vue
<!-- 错误：复制自 Linksinfomgmt.vue -->
<NewsButton perms="['web:linksinfomgmt:view', ...]"/>
```

**修复**：

```js
// utils/perms.js
export const PERMS = {
  USER_VIEW: 'sys:user:view',
  USER_ADD: 'sys:user:add',
  LINKS_VIEW: 'web:linksinfomgmt:view',
  PASSIP_VIEW: 'web:passipinfomgmt:view',
  // ...
}
```

```vue
<NewsButton :perms="[PERMS.PASSIP_VIEW, ...]"/>
```

### 老朋友 4：登录响应判断

**症状**：

```js
if (res.msg === '交易成功') {  // ⚠️ 字符串依赖
  // ...
}
```

**修复**：

```js
if (res.code === 200) {
  // ...
}
```

### 老朋友 5：硬编码 baseUrl

**症状**：

```js
const baseUrl = 'http://10.52.9.110:9091/api'  // ⚠️
```

**修复**：

```js
const baseUrl = process.env.VUE_APP_API_BASE || `${location.origin}/api`
```

## 6. 三个关键金句

> **金句 1**：改代码 = 改资产；改资产 = 改代码。资产是代码的伴生品而非附件。

> **金句 2**：P0 > 10 条 = 没分级。严重度体系失效比没体系更危险。

> **金句 3**：AOP 切面是最容易被遗忘的"半成品"——代码在，但永远不触发。所有"被动机制"（AOP、定时任务、事件监听）必须有"已触发"测试。
