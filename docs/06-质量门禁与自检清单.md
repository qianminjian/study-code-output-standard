# 06 · 质量门禁与自检清单

> 资产发布前必跑 8 类自检。**未通过 = 不允许合并**。

---

## 总览：8 类自检

| # | 类型 | 自动化程度 | 责任方 |
|:-:|---|:-:|---|
| 1 | 元信息头 | 100% 自动 | CI |
| 2 | 严重度字段 | 100% 自动 | CI |
| 3 | 资产-代码一致性 | 80% 自动 | CI + 人工抽 |
| 4 | 资产间一致性 | 50% 自动 | CI + 人工 |
| 5 | 链接有效性 | 100% 自动 | CI |
| 6 | 严重度合理性 | 50% 自动 | 评审 |
| 7 | 反模式扫描 | 80% 自动 | CI |
| 8 | AI 可消费性 | 30% 自动 | 评审 |

---

## 1. 元信息头自检

**目的**：确保每篇资产可追溯。

### 1.1 校验项

- [ ] `id` 字段值 = 文件名去后缀
- [ ] `version` 字段为 SemVer
- [ ] `last_updated` 字段为 YYYY-MM-DD
- [ ] `data_source` 字段不为空
- [ ] `code_version` 字段为 git hash 或 tag
- [ ] `owner` 字段不为空
- [ ] `ai_consumable` 字段为 `true/false`

### 1.2 自动脚本

```bash
#!/usr/bin/env bash
# scripts/check-meta.sh
set -e
REQUIRED=("id" "version" "last_updated" "data_source" "code_version" "owner" "ai_consumable")
for f in docs/*.md; do
  for k in "${REQUIRED[@]}"; do
    grep -q "\"$k\":" "$f" || { echo "MISSING META $k in $f"; exit 1; }
  done
done
echo "OK"
```

## 2. 严重度字段自检

**目的**：确保 P0-P3 体系严格。

### 2.1 校验项

- [ ] 所有严重度值 ∈ {P0, P1, P2, P3}
- [ ] 严禁出现 `P4+` 或自定义（"中"、"高"、"严重"）
- [ ] emoji 前缀统一（🔴 P0 / 🟡 P1 / 🟢 P2 / ⚪ P3）
- [ ] 严重度在表格中列名固定

### 2.2 自动脚本

```bash
#!/usr/bin/env bash
# scripts/check-severity.sh
set -e
INVALID=$(grep -E "P[4-9]|[严重高中低]等" docs/ -r | head -5)
[ -z "$INVALID" ] || { echo "INVALID SEVERITY: $INVALID"; exit 1; }
echo "OK"
```

## 3. 资产-代码一致性

**目的**：确保资产**反映代码事实**。

### 3.1 校验项

| 资产 | 验证命令 | 期望 |
|---|---|---|
| 03-API 清单 | `grep -rE "@(Get\|Post\|Put\|Delete)Mapping" src/ | wc -l` | = 03 表中端点总数 |
| 03-Controller 概览 | `find src/ -name "*Controller.java" \| wc -l` | = 03 概览表行数 |
| 04-Mapper 清单 | `find src/ -name "*Mapper.java" \| wc -l` | = 04 概览表行数 |
| 04-XML 数 | `find resources/mybatis -name "*.xml" \| wc -l` | = 04 XML 数 |
| 02-Model 数 | `find src/ -name "*.java" -path "*/model/*" \| wc -l` | = 02 Model 数 |
| 02-表数 | `grep -c "CREATE TABLE" db-scripts/*.sql` | = 02 表数 |
| 07-组件数 | `find src/{views,components} -name "*.vue" \| wc -l` | = 07 组件总数 |
| 08-Store 模块 | `ls src/store/modules/ \| wc -l` | = 08 Store 模块数 |

### 3.2 通用脚本骨架

```bash
#!/usr/bin/env bash
# scripts/check-consistency.sh
set -e

# 1. 端点
expected=$(grep -rE "@(Get|Post|Put|Delete)Mapping" src/ | wc -l)
documented=$(grep -cE "^\| (GET|POST|PUT|DELETE) \|" docs/03-*.md || true)
[ "$expected" -eq "$documented" ] || { echo "MISMATCH: endpoints $expected vs $documented"; exit 1; }

# 2. Controller
expected=$(find src/ -name "*Controller.java" | wc -l)
documented=$(awk '/^### 2\./{c++} END{print c}' docs/03-*.md)
[ "$expected" -eq "$documented" ] || { echo "MISMATCH: controllers"; exit 1; }

# 3. ... 更多

echo "OK"
```

### 3.3 人工抽样

- 抽 5-10 个端点对照 Swagger / Postman
- 抽 3-5 个业务流走一遍
- 抽 10 条 P0 验证行号存在

## 4. 资产间一致性

**目的**：确保 12 篇资产**互相对得上**。

### 4.1 强引用对账

| 关系 | 检查方式 |
|---|---|
| 02-数据模型 字段 ↔ 03-API 入参 | 03 提到 X 实体 → 02 必有 X 表 |
| 03-API 鉴权 ↔ 06-安全放行路径 | 03 标"公开" → 06 必须在放行列表 |
| 03-API 端点 ↔ 07/08 前端调用 | 07/08 提到 `/api/X` → 03 必有 |
| 05-业务流 ↔ 10-业务流图 | 05 提到的流编号 → 10 必含 |
| 11-技术债 ↔ 12-修复建议 | 12 每条 → 11 必有对应 |

### 4.2 自动校验

```python
# scripts/check-cross-ref.py
import re
import os
import sys

# 1. 加载所有 API 端点
endpoints = set()
for line in open("docs/03-后端-Controller接口清单.md"):
    m = re.search(r"\| `/api/(\w+/\w+)` \|", line)
    if m: endpoints.add(m.group(1))

# 2. 加载所有前端 API 调用
used = set()
for line in open("docs/07-前端-页面与组件清单.md"):
    m = re.search(r"/api/(\w+/\w+)", line)
    if m: used.add(m.group(1))

# 3. 找出前端调用但后端没有的（孤儿）
orphans = used - endpoints
if orphans:
    print(f"ORPHAN API CALLS: {orphans}")
    sys.exit(1)

# 4. 找出后端有但前端不用的（可考虑删除）
unused = endpoints - used
if unused:
    print(f"UNUSED ENDPOINTS: {unused}")

print("OK")
```

### 4.3 人工校验

- 走查 02↔03、03↔07、05↔10、11↔12 四对强引用
- 每个引用关系至少验证 3 个具体点

## 6. 严重度合理性

**目的**：避免"全是 P0"或"全是 P3"。

### 6.1 分布期望

| 等级 | 期望占比 | 期望总数（中型项目） |
|---|---|---|
| P0 | < 10% | 0-10 条 |
| P1 | 20-30% | 10-30 条 |
| P2 | 30-40% | 20-50 条 |
| P3 | 30-40% | 20-50 条 |

### 6.2 校验项

- [ ] P0 总数 < 10 条（**多了 = 没分级**）
- [ ] P0 每条都有 24h 内可执行的修复方案
- [ ] P1 不少于 P0
- [ ] P3 不少于 P2（**少了 = 死代码没清**）

### 6.3 评审

- 走查 P0/P1 各 3 条，确认严重度合理
- 评审会确认 11/12 严重度

## 7. 反模式扫描

**目的**：把已知反模式显式标出。

### 7.1 必扫反模式

| 反模式 | 扫法 |
|---|---|
| `@sqlinjection` | `grep -rE "\\$\\{[^}]+\\}" src/main/resources/mybatis/` |
| `@secret-leak` | `grep -rE "secret.*=.*[\"'][^\"']{8,}[\"']" src/main/java/` |
| `@cors-wildcard` | `grep -r "allowedOrigins(\"\\*\")" src/main/java/` |
| `@hardcoded` | `grep -rE "(127\\.0\\.0\\.1\|localhost\|10\\.\\d+\\.\\d+\\.\\d+)" src/` |
| `@admin-bypass` | `grep -rE "admin" src/main/java/` + 人工 |
| `@xss` | `grep -rE "\\.html\\([^)]*content[^)]*\\)" src/` |
| `@missing-tx` | `grep -rL "@Transactional" src/main/java/**/service/impl/` |
| `@todo-stub` | `grep -r "TODO Auto-generated" src/main/java/` |

### 7.2 脚本骨架

```bash
#!/usr/bin/env bash
# scripts/scan-antipatterns.sh
echo "=== @sqlinjection ==="
grep -rnE "\\$\\{[^}]+\\}" src/main/resources/mybatis/ || echo "  (clean)"

echo "=== @secret-leak ==="
grep -rnE "(secret|password|token)\\s*=\\s*[\"'][^\"']{8,}[\"']" src/main/java/ \
  | grep -v "test" || echo "  (clean)"

echo "=== @cors-wildcard ==="
grep -rn "allowedOrigins(\"\\*\")" src/main/java/ || echo "  (clean)"

echo "=== @hardcoded IP/host ==="
grep -rnE "(127\\.0\\.0\\.1|10\\.[0-9]+\\.[0-9]+\\.[0-9]+)" src/main/java/ || echo "  (clean)"

echo "=== @todo-stub ==="
grep -rn "TODO Auto-generated" src/main/java/ | wc -l
```

### 7.3 自动入档

把扫出的反模式**直接追加**到 11-技术债清单草稿。

## 9. CI 接入

### 9.1 GitHub Actions 示例

```yaml
# .github/workflows/docs-validate.yml
name: Docs Validate
on:
  pull_request:
    paths:
      - 'docs/**'
      - 'src/**'
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check Meta
        run: bash scripts/check-meta.sh
      - name: Check Severity
        run: bash scripts/check-severity.sh
      - name: Check Consistency
        run: bash scripts/check-consistency.sh
      - name: Check Links
        run: npx markdown-link-check docs/*.md
      - name: Scan Antipatterns
        run: bash scripts/scan-antipatterns.sh
```

### 9.2 失败处理

CI 失败 → PR 阻断 → 修复资产或代码 → 重跑。
