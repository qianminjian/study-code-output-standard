# AI Prompt · 02 数据模型

## 完整 Prompt

```
【背景】
项目 <name>，技术栈 <list>。本任务生成"数据模型与表结构"资产。

【任务】
生成 docs/02-数据模型与表结构.md。
严格按 docs/03-文档模板与质量标准.md §02 模板。

【输入】
- ORM 实体目录：<path>（如 src/main/java/.../model/）
- Mapper XML 目录：<path>（如 src/main/resources/mybatis/）
- 数据库脚本：<path>（如 db-scripts/*.sql）
- 关注：继承关系（BaseModel）、字段类型（Long vs String）、命名（小驼峰 vs 蛇形）

【输出字段】
1. 实体类与表对应关系（表：Model、表、继承、关键字段、备注）
2. 数据库表清单（表：表名、中文、主要字段、索引/约束）
3. 关键关联关系（ER 图 ASCII / Mermaid + 字段映射约定）
4. MyBatis XML 命名空间总览（表：XML、namespace、操作）
5. 关键 SQL 模式（分页、Join、resultMap、注入风险）
6. ⚠️ 已知数据模型问题（如命名不一致、id 类型不一致）

【强制列】
- 实体表：| Model | 表 | 继承 | 关键字段 | 备注 |
- 表清单：| 表名 | 中文 | 主要字段 | 索引/约束 |

【SQL 注入扫描】
必跑：`grep -rE "\\$\\{[^}]+\\}" <xml-dir>` 找出 `${}` 字符串拼接。

【反例】
- 不漏 BaseModel
- 不漏继承关系
- 不漏字段映射错位（对比 Model 和 XML column）
- 不漏 SQL 注入风险
- 不漏表名命名不一致

【自我校验】
- Model 数 = `find <model-dir> -name "*.java" | wc -l`
- XML 数 = `find <xml-dir> -name "*.xml" | wc -l`
- 表数 = `grep -c "CREATE TABLE" <sql-files>`
- 注入数 = `grep -rE "\\$\\{[^}]+\\}" <xml-dir> | wc -l`
- 输出问题数与扫描结果一致
```
