# AI Prompt · 05 服务与业务逻辑

## 完整 Prompt

```
【背景】
项目 <name>，后端 <栈>。本任务生成"服务与业务逻辑"资产。

【任务】
生成 docs/05-后端-服务与业务逻辑.md。
按 docs/03-文档模板与质量标准.md §05 模板。

【输入】
- Service 接口目录：<path>
- ServiceImpl 目录：<path>
- 关注：跨 Mapper 协作、关键算法、事务边界、异常处理、admin 保护

【输出字段】
1. 接口与实现对应（表：Service、Impl、注入 Mapper、业务范围）
2. 核心业务流（每个流：输入→步骤→输出→边界→风险）
3. ⚠️ 已知 Service 实现问题

【关键业务流（必备）】
- 密码加盐 / 验密
- 超级管理员保护
- 菜单树 / 部门树 / 栏目树递归
- 业务单据创建（含查重）
- 业务单据查询（含点击数累加）
- 改密 / 重置密码
- 字典 / 配置查询
- 文件上传 / 导出
- IP 白名单比对

【每个业务流包含】
- 名称 + 涉及类
- 输入（参数）
- 步骤（伪代码或文字步骤）
- 输出（返回）
- 边界（异常、空、超长）
- ⚠️ 风险（如有）

【反例】
- 不写"调用 X 然后 Y" 的纯流水账
- 不漏事务边界
- 不漏 admin 保护逻辑
- 不漏接口未实现的 Impl 方法

【自我校验】
- Service 数 = `find <service-dir> -name "*.java" | wc -l`
- Impl 数 = `find <impl-dir> -name "*.java" | wc -l`
- 业务流 ≥ 6
- 元信息头 7 字段
```
