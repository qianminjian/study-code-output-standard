# AI Prompt · 01 系统总览

> ⚠️ **顺序警示**：01 必须**最后写**（依赖 02-12 全部就位）。即使叫"系统总览"，也应在所有子资产完成后才生成。
> 抽完 02 → 12 后再调此 Prompt，否则 AI 写出来必空。
> 适用：任何项目反向阅读的最后一步。
> 三段式：背景 + 任务 + 反例。

---

## 完整 Prompt

```
【背景】
你正在为一个 <技术栈> 的 <项目类型> 项目写"系统总览"资产。
代码根目录：<path>。
整理时间：<YYYY-MM-DD>。

【任务】
生成 docs/01-系统总览.md。
严格按 docs/03-文档模板与质量标准.md §01 模板。

【输入】
- 跑 `tree -L 3 <path>` 拿目录结构
- 读 README.md / package.json / pom.xml / requirements.txt / go.mod
- 找启动入口文件（main / Application / index.js / app.py）
- 跑 `find <path> -name "*Controller*" -o -name "*Service*" -o -name "*Mapper*" -o -name "*.vue" -o -name "*.tsx" | wc -l` 拿数字
- 跑 `grep -rE "@(Get|Post|Put|Delete)Mapping" <path> | wc -l` 拿端点数
- 跑 `grep -rE "TODO|FIXME" <path> | wc -l` 拿 TODO 数

【输出字段】
1. 项目背景（2-3 句：业务定位、目标用户）
2. 项目边界（目录树 markdown 块）
3. 技术栈（后端 / 前端 / 静态端 三个表 + 版本号）
4. 多端关系（ASCII 架构图，含端口与协议）
5. 关键路径（按子域列出文件位置）
6. 启动顺序（Step-by-Step，每步可执行命令）
7. 数字一览（Controller / Service / Model / 端点 / 组件 / 权限串 / TODO）

【强制要求】
- 元信息头 frontmatter 完整（7 字段）
- 顶部引用块：> 整理时间 · 数据来源
- 数字一览必须可验证（每行后注验证命令）
- 不超过 1000 行

【反例】
- 不写"该项目使用 X 框架"而不写版本
- 不写散文式"系统特点"段落
- 不写"约 70 个端点"而 grep 数清楚
- 不省启动命令

【自我校验】
生成后请：
1. 跑 `find docs/ -name "01-*.md" -exec wc -l {} \;` 不超 1000
2. 数字一览每行都有验证命令
3. 元信息头 7 字段齐全
4. 启动顺序每步可执行
```

---

## 使用方法

1. 复制上方 Prompt
2. 替换 `<...>` 占位符
3. 喂给 AI（Claude Code / Cursor / Continue）
4. AI 输出后人工校验

## 校验清单

- [ ] 元信息头 7 字段齐全
- [ ] 数字一览与 grep 一致
- [ ] 启动顺序可执行
- [ ] 架构图清晰
- [ ] 引用块格式正确
