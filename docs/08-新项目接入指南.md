# 08 · 新项目接入指南（v2 — Skill 模式）

> 通过 `study-code-output-standard` **Skill 封装** 自动化反向阅读任意项目。
> 一次安装，所有项目可用；输出到 `${TARGET}/asset-docs/`，CLAUDE.md + CLAUDE-ASSET.md 自动生成。

---

## 1. 三种使用方式（v2：统一为 Skill）

### 方式 A：个人全局安装（推荐）

```bash
# 一次性安装
bash <methodology-repo>/skill/install.sh --personal

# 在任意项目使用
cd /path/to/target-app
claude
> /study-code-output-standard
```

### 方式 B：项目级安装（团队共享）

```bash
cd /path/to/target-app
bash <methodology-repo>/skill/install.sh --project
# 效果：在 .claude/skills/ 创建软链
```

### 方式 C：指定路径（测试/开发）

```bash
bash <methodology-repo>/skill/install.sh --path /some/install/location
```

> **方法论升级时**（安装的是软链），所有项目自动跟随新版。

---

## 2. Quickstart（5 步上手）

```bash
# === 步骤 1：安装 Skill（一次性）===
bash /opt/study-code-output-standard/skill/install.sh --personal

# === 步骤 2：进入目标项目 ===
cd /path/to/target-app

# === 步骤 3：启动 Claude Code ===
claude

# === 步骤 4：调用 Skill ===
# 在 Claude Code 中：
> /study-code-output-standard

# AI 自动按 6 步法执行（勘探→分类→抽取→校验→标注→派送）
# 输出到 ./asset-docs/

# === 步骤 5：人工 review + 接入 CI ===
bash asset-docs/scripts/validate-all.sh
# 接入 CI 见 06-质量门禁 §9
```

### 3.5 人工 review

抽 5 个点验证：

1. **CLAUDE.md** ≤ 80 行，标"按需加载"
2. **CLAUDE-ASSET.md** 资产状态正确（占位/✓）
3. **01-总览** 数字与 grep 对齐
4. **03-API** 端点 = `grep -c "@\(Get\|Post\)Mapping" src/main/java`
5. **06-安全** 密钥/密码 已 <REDACTED>

### 3.6 接入 CI

`.github/workflows/docs-validate.yml`：

```yaml
name: Docs Validate
on:
  pull_request:
    paths:
      - 'asset-docs/**'
      - 'src/**'
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run all checks
        run: bash asset-docs/scripts/validate-all.sh
```

## 4. CLAUDE.md 双文件策略

| 文件 | 行数 | 用途 | 加载方式 |
|---|---|---|---|
| `CLAUDE.md` | ≤ 80 | 轻量索引 + 工作流 + 反模式 | Claude 启动时自动加载 |
| `CLAUDE-ASSET.md` | 100-200 | 12 篇资产详细地图 + 强引用关系 | 按需 Read |
| `asset-docs/<NN>-*.md` | 不限 | 单篇资产详情 | 按需 Read |

**双文件原则**：
- CLAUDE.md 不塞资产详情（避免上下文超限）
- CLAUDE-ASSET.md 标"按需加载"（不强制 Read）
- 单篇资产标"具体场景下 Read"

## 5. 进阶用法

### 5.1 让 AI 写新功能

**最小喂入（3 篇）**：

```
请按 asset-docs/01-系统总览.md、02-数据模型与表结构.md、03-后端-Controller接口清单.md 的规范，
新增一个 POST /api/orders/create 接口。
```

**推荐喂入（5 篇）**：+ `05-后端-服务与业务逻辑.md` + `11-技术债与遗留项.md`

**完整喂入（12 篇）**：可承担 新功能 / 重构 / 修 Bug。

### 5.2 让 AI 修 P0

```
请按 asset-docs/11-技术债与遗留项.md 中 P0 #1 "JWT 密钥硬编码" 修复：
1. JwtTokenUtils 改读 ${jwt.secret}
2. 启动校验非默认值
3. 加单元测试
4. 同步更新 asset-docs/06-后端-安全认证.md
5. 同步更新 asset-docs/12-修复建议与优先级.md（标记 #1 已修复）
```

### 5.3 资产演进

修改代码的 PR 必走：

```bash
# 1. 改代码
# 2. 同步改 asset-docs/ 对应章节
# 3. 跑 asset-docs/scripts/validate-all.sh
# 4. 在 asset-docs/CHANGELOG.md 加一行
# 5. 提交
```

## 7. 故障排查

| 症状 | 原因 | 修复 |
|---|---|---|
| `claude` 不识别 skill | install 没成功 | `bash skill/install.sh --personal` |
| 软链创建失败 | Windows 非 Git Bash | 改用 PowerShell `install.ps1` |
| AI 输出 12 篇但全是模板 | Prompt 没喂全 | 检查 ai-prompts/00-全流程启动.md 完整粘贴 |
| 数字与代码不一致 | AI 没 grep | Prompt 强制 grep 输出 |
| 密钥/密码出现在 06-安全 | AI 复制了原文 | Prompt 加"必须 <REDACTED>" |
| 校验脚本报 P0 > 10 | 模板占位符混入 | init 完后把 templates/ 排除，或手动填后再校验 |
| 路径找不到方法论 | install 软链断了 | 重新 `bash install.sh --personal` |
