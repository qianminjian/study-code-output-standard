# AI Prompt · 11 技术债清单

## 完整 Prompt

```
【背景】
项目 <name>，01-10 资产已生成。本任务生成"技术债与遗留项"资产。

【任务】
生成 asset-docs/11-技术债与遗留项.md。
按 methodology/03-文档模板与质量标准.md §11 模板。

【输入】
- 已生成的 01-10 资产
- 全仓库扫描：TODO/FIXME/注释代码/硬编码/死代码

【输出字段】
1. 注释代码块（应清理）
2. 命名不一致
3. 死代码（应删除）
4. TODO/FIXME 清单
5. 硬编码（应外置）
6. 注解 / 标签错位
7. 权限字符串错用
8. 性能问题
9. 版本 / 生态落后
10. 文档与代码不一致

【强制列】
| 严重度 | 类别 | 位置 | 问题 | 建议 |
| 🔴P0 / 🟡P1 / 🟢P2 / ⚪P3 | <类别> | <文件:行> | <问题> | <建议> |

【自动扫描（必跑）】
- 注入：`grep -rE "\\$\\{[^}]+\\}" <xml-dir>`
- 密钥：`grep -rE "(secret|password|token)\\s*=\\s*[\"'][^\"']{8,}[\"']" <java-dir>`
- CORS：`grep -rn "allowedOrigins(\"\\*\")" <java-dir>`
- 硬编码：`grep -rnE "(127\\.0\\.0\\.1|10\\.[0-9]+\\.[0-9]+\\.[0-9]+)" <src>`
- TODO：`grep -rE "(TODO|FIXME|XXX)" <src> | wc -l`
- 死代码：人工识别

【反模式标签（必带）】
@sqlinjection @secret-leak @cors-wildcard @perm-mismatch @admin-bypass
@null-pointer @n+1 @dead-code @hardcoded @wrong-default
@xss @csrf @missing-tx @todo-stub @copy-paste
@magic-number @long-function @long-param
@feature-envy @shotgun-surgery @wrong-package @wrong-tag
@missing-i18n @wrong-token-hdr

【反例】
- 不写"建议重构" 而不给具体方案
- 不写"性能有问题" 而不给行号
- 不写自定义严重度（"中"、"高"）
- 不漏位置

【自我校验】
- P0 < 10 条
- P1 不少于 P0
- P3 不少于 P2
- 总数 > 20
- 元信息头 7 字段
```
