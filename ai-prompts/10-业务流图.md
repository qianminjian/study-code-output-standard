# AI Prompt · 10 业务流图

## 完整 Prompt

```
【背景】
项目 <name>，02/03/05/07/08 资产已生成。本任务生成"端到端业务流图"资产。

【任务】
生成 asset-docs/10-业务流图.md。
按 methodology/03-文档模板与质量标准.md §10 模板。

【输入】
- 已生成的 02/03/05/07/08 资产
- 关注：跨层调用链、关键决策点、异常分支

【输出字段】
每个核心业务流一节：
- 触发条件
- 完整调用链（ASCII 流程图）
- 关键决策点
- 相关代码位置（文件 + 行号）
- 已知流相关 Bug

【必备业务流（至少 6 个）】
1. 主登录流（后台 + 前台）
2. 主业务创建流
3. 主业务查询流
4. 权限校验流
5. 文件上传 / 导出流
6. 改密 / 找回密码流

【ASCII 流程图规范】
```
[用户] 打开 <URL>
  → 路由守卫
  → 跳 /login
[Login.vue] mounted
  → refreshCaptcha()
  → ...
[后端 Controller] login
  → 校验 captcha
  → 验密
  → ...
  → 返回 JWT
[前端]
  → 存 token
  → 跳首页
```

【反例】
- 不写"调了 service" 而不写 service 名
- 不漏异常分支
- 不漏关键决策点（如 IP 白名单比对、admin 保护）
- 不漏文件位置

【自我校验】
- 业务流 ≥ 6
- 业务流编号与 05 对齐
- 元信息头 7 字段
```

## 校验清单

- [ ] 6+ 核心流
- [ ] 每个流含触发→前端→后端→落库→响应
- [ ] 异常分支有标注
- [ ] 业务流编号与 05 对齐
