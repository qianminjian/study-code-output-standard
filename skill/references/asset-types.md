## 各资产必填字段速查

### 02-数据模型

| 必填 | 说明 |
|:-:|---|
| ✅ | Model 数量 = 实际 `model/*.java` 数量 |
| ✅ | 表数量 = 实际 `*.sql` 数量 |
| ✅ | 每个 Model 有"对应表" |
| ✅ | ER 图含主外键关系 |
| ✅ | 标注 SQL 注入风险 |

### 03-API 清单

| 必填 | 说明 |
|:-:|---|
| ✅ | 端点总数 = `@(Get\|Post\|Put\|Delete)Mapping` 数量 |
| ✅ | 路径含全局前缀 |
| ✅ | 每个端点有鉴权标注 |
| ✅ | 公开端点单独列 |
| ✅ | 权限串全集与代码一致 |

### 04-Mapper 清单

| 必填 | 说明 |
|:-:|---|
| ✅ | Mapper 数 = 实际 mapper 文件数 |
| ✅ | XML 数 = 实际 mybatis/*.xml 数 |
| ✅ | 每个方法标注业务用途 |
| ✅ | SQL 模式至少含 3 类 |

### 05-服务与业务

| 必填 | 说明 |
|:-:|---|
| ✅ | 业务流覆盖主流程 |
| ✅ | 每个业务流含输入/输出/边界 |
| ✅ | 业务流编号与 10 对齐 |

### 06-安全认证

| 必填 | 说明 |
|:-:|---|
| ✅ | 安全风险与 OWASP Top 10 交叉检查 |
| ✅ | 密钥/密码用 `<REDACTED>` 替代 |
| ✅ | 多环境差异表完整 |

### 07-前端页面

| 必填 | 说明 |
|:-:|---|
| ✅ | 组件数 = 实际 `views + components` 文件数 |
| ✅ | 死代码单列 |
| ✅ | 权限与 03 对齐 |

### 08-前端状态

| 必填 | 说明 |
|:-:|---|
| ✅ | Store 模块数 = 实际文件数 |
| ✅ | 动态路由映射表完整 |
| ✅ | API 模块与 03 对齐 |

### 10-业务流图

| 必填 | 说明 |
|:-:|---|
| ✅ | 6+ 核心业务流 |
| ✅ | 每个流含触发→前端→后端→落库→响应 |
| ✅ | 异常分支有标注 |

### 11-技术债

| 必填 | 说明 |
|:-:|---|
| ✅ | 每个 P0/P1 能追溯到代码位置 |
| ✅ | 不出现"建议重构"等无具体方案 |
| ✅ | 数量 > 20 |
| ✅ | 严重度 P0 < 10% |

### 12-修复建议

| 必填 | 说明 |
|:-:|---|
| ✅ | 每条都能追溯到 11 |
| ✅ | P0 全部有具体修改方案 |
| ✅ | 整体重构路径分阶段 |
