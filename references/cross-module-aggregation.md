# v3.5 跨模块聚合协议

> 本文件定义编排者在所有 worker 完成后的**聚合阶段**产出规范。
> 触发时机：Phase 1-4 全部 worker 产出完成后，编排者进入聚合阶段。
> 产物：`asset-docs/00-CROSS-MODULE.md` + `asset-docs/00-CROSS-SECURITY.md`
> 执行者：**编排者自己**（变体 D: Aggregate），不 spawn worker。

---

## 1. 聚合触发条件

| 条件 | 说明 |
|---|---|
| 模块数 ≥ 2 | 单模块项目跳过（无"跨"可言） |
| Phase 1-4 全部完成 | 所有资产文件（02-13）均存在于 `asset-docs/` |
| 01-系统总览已产出 | 编排者在 Phase 4 后已写好 01 |

**跳过场景**：纯后端模块但前端跳过（07/08/09 不存在）不属于跳过条件——跨模块聚合仍有价值（跨后端各模块的一致性校验）。

---

## 2. 聚合输入（必读资产）

编排者在聚合前**必须通读**以下全部资产（变体 D 铁律）：

| 输入 | 用途 |
|------|------|
| 各模块的 `01-系统总览.md` | 提取技术栈、目录结构、数字一览 |
| `02-数据模型与表结构.md` | 提取 ⚠️ 推断表、Model 数 |
| `03-后端-Controller接口清单.md` | 提取端点数、公开端点数、鉴权策略 |
| `04-后端-Mapper操作清单.md` | 提取 Mapper 数、SQL 注入标记 |
| `05-后端-服务与业务逻辑.md` | 提取业务规则、调用链 |
| `06-后端-安全认证.md` §7 | 提取安全风险清单（P0/P1） |
| `11-技术债与遗留项.md` | 提取跨模块技术债 |
| `13-反模式扫描报告.md` | 提取反模式（去重用） |

> ⚠️ **变体 D 铁律**：通读全部上游资产，不能只读摘要行。如果只读 §1 摘要表而跳过正文，产出无效。
