## 2. Spring Security 配置

### 2.1 WebSecurityConfig

```java
@Configuration
@EnableWebSecurity
public class WebSecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .csrf().disable()                    // ⚠️ CSRF 禁用
            .cors()                              // CORS 配置
            .and()
            .authorizeRequests()
                .antMatchers("/oauth/**").permitAll()  // 公开端点
                .antMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            .and()
            .sessionManagement()
                .sessionCreationPolicy(STATELESS)  // 无状态 JWT
            .and()
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);
    }
}
```

| 配置项 | 值 | 风险评估 |
|---|---|---|
| csrf().disable() | 已禁用 | ⚠️ POST/PUT/DELETE 无 CSRF 保护 |
| cors() | 未配置 | ⚠️ 需检查 CorsConfigurationSource |
| sessionCreationPolicy | STATELESS | ✅ JWT 无状态认证 |
| addFilterBefore | JWT Filter 优先 | ✅ |

### 2.2 认证全流程

```
[用户] 登录请求
  → JwtAuthenticationFilter.doFilter()
  → 从 Header 提取 token (Authorization: Bearer xxx)
  → JwtTokenUtil.validateToken()
  → 解析 token 获取用户信息
  → SecurityContextHolder.setAuthentication()
  → 后续请求带用户权限

[失败路径]
  → Token 无效/过期
  → JwtAuthenticationFilter 返回 401
  → 客户端跳转登录页
```

## 3. JWT Token 分析（v3.0 核心）

**Token 生成**：
```java
public String generateToken(UserDetails userDetails) {
    Map<String, Object> claims = new HashMap<>();
    claims.put("roles", userDetails.getAuthorities());
    claims.put("permissions", getPermissions(userDetails));

    return Jwts.builder()
        .setClaims(claims)
        .setSubject(userDetails.getUsername())
        .setIssuedAt(new Date())
        .setExpiration(new Date(System.currentTimeMillis() + expiration))
        .signWith(key, SignatureAlgorithm.HS256)
        .compact();
}
```

## 4. 权限控制链

```
[请求] GET /api/user/123
  → JwtAuthenticationFilter 验证 Token
  → SecurityContextHolder 获取用户
  → @PreAuthorize("hasPermission(#id, 'user', 'view')")
  → 自定义 PermissionEvaluator.evaluate()
  → 返回 200 / 403
```

| 权限控制方式 | 实现 | 位置 |
|---|---|---|
| @PreAuthorize | SpEL 表达式 | Controller 方法 |
| @Secured | 角色检查 | Service 方法 |
| hasPermission() | 自定义评估器 | PermissionEvaluator |
| 硬编码判断 | if-else | 代码中 |

## 5. AOP 切面分析

### 5.1 审计日志切面

```java
@Aspect
@Component
public class AuditLogAspect {

    @Pointcut("execution(* com.example..controller.*.*(..))")
    public void controllerPointcut() {}

    @Around("controllerPointcut()")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        // 记录操作
        LogEntity log = new LogEntity();
        log.setMethod(joinPoint.getSignature().toShortString());
        log.setArgs(joinPoint.getArgs());
        log.setOperator(getCurrentUser());

        long start = System.currentTimeMillis();
        Object result = joinPoint.proceed();
        long duration = System.currentTimeMillis() - start;

        log.setDuration(duration);
        logService.save(log);

        return result;
    }
}
```

| 切面 | 切入点 | 通知类型 | 问题 |
|---|---|---|---|
| AuditLogAspect | Controller | @Around | 🔴 Pointcut 写错包名未触发 |
| SysLogAspect | @LogAnnotation | @Before | ⚠️ 注解方法不存在 |
| PerformanceAspect | 全部方法 | @Around | 🟡 性能监控 |

### 5.2 死切面识别

| 切面 | 预期切入点 | 实际情况 | 状态 |
|---|---|---|---|
| AuditLogAspect | com.example.controller.* | 包名不存在 | 🔴死切面 |
| PerformanceAspect | *Service.*(..) | 实际切了全部 | 🟡性能影响 |

## 6. CORS 配置分析

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
            .allowedOrigins("*")                    // ⚠️ P0 CORS
            .allowedMethods("GET", "POST", "PUT", "DELETE")
            .allowedHeaders("*")
            .allowCredentials(true)                // ⚠️ 与 * 冲突
            .maxAge(3600);
    }
}
```

| 配置项 | 值 | 风险 | 严重度 |
|---|---|---|---|
| allowedOrigins | * | 任意域可访问 | 🔴P0 |
| allowCredentials | true | 与 * 冲突 | 🔴P0 |
| allowedMethods | GET,POST,PUT,DELETE | — | — |
