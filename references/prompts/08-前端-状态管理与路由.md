# AI Prompt · 08 前端状态与路由

## 完整 Prompt

```
【背景】
项目 <name>，前端框架 <Vue/React/...>，状态管理 <Vuex/Pinia/Redux/...>。本任务生成"状态管理与路由"资产。

【任务】
生成 asset-docs/08-前端-状态管理与路由.md。
按 references/methodology/03-文档模板与质量标准.md §08 模板。

【输入】
- store/ 目录
- router/ 目录
- http/ 或 api/ 目录
- utils/ 目录
- main 配置

【输出字段】
1. 状态管理模块（state / mutations / actions / getters）
2. 路由（静态 + 动态 + 映射表）
3. API 模块清单
4. HTTP 客户端封装（axios / fetch 拦截器）
5. i18n / 工具
6. ⚠️ 已知问题

【关键发现】
- request 拦截器对所有请求校验 token（登录死循环）
- token header 用非标准（如直接 `token:` 而非 `Authorization: Bearer`）
- setPerms 在错误模块（导致按钮权限失效）
- 每次请求 new axios instance
- 多语言缺键

【反例】
- 不漏 setPerms 位置
- 不漏 token header 非标准
- 不漏 HTTP 客户端拦截器 bug
- 不漏 i18n 缺键

【自我校验】
- Store 模块数 = `ls <store>/modules/ | wc -l`（Vuex）或 `find <store> -name "*.ts" -o -name "*.js" | wc -l`
- API 模块数 = `ls <api> | wc -l`
- 元信息头 7 字段
```

## 校验清单

- [ ] Store 模块数与文件数一致
- [ ] 动态路由映射表完整
- [ ] API 模块与 03 对齐
