## 7. 启动顺序

1. **启动基础设施**：MySQL / 注册中心
2. **启动 UPMS**：统一权限服务（8080）
3. **启动 Gateway**：API 网关
4. **启动业务模块**：platform / pipeline / flow 等
5. **启动前端**：npm run dev
6. 访问：`http://localhost:8000`

| 服务 | 依赖 | 单点故障风险 |
|---|---|---|
| API Gateway | UPMS | 高（无 Token 全站不可用）|
| 业务模块 | UPMS + Registry | 中（部分功能降级）|
