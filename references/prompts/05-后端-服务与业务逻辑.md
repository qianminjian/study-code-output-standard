## 3. 核心业务流

### 3.1 登录认证流

**完整路径**：
```
[用户] 输入用户名/密码
  → POST /oauth/token (ExternalAuthController)
  → HashUtil.decodeBase64() 解密密码
  → LoginAuthentication.checkPassword() 验证
  → Oauth2Service.generateToken() 生成 token
  → 返回 { token, refreshToken }
```

**决策点分析**（v3.0 核心）：
| 决策点 | 当前决策 | 推断依据 | 合理性 |
|---|---|---|:---:|---|
| 密码解密位置 | Controller 层 | ExternalAuthController:204 | ⚠️ 存疑：应上移至 Service |
| 密码错误计数 | 未确认 | 无显式 loginErrorCount 重置 | ⚠️ 可能导致误锁 |
| Token 存储位置 | 前端 Vuex | Header 读取 | ✅ 合理：支持多标签页 |

**异常路径**（v3.0 核心）：
| 异常场景 | 当前处理 | 期望处理 | 差距 |
|---|---|---|---|
| 密码错误 | USERNAME_OR_PASSWORD_FAILED | 同左（防枚举） | ✅ |
| 账户锁定 | ACCOUNT_LOCKED_ERROR | 同左 | ✅ |
| 密码为空 | 未校验 | 应拒绝空密码 | 🔴差距 |
| Token 过期 | TOKEN_EXPIRED | 自动刷新 | 🟡差距 |

**隐含假设**：
| 假设 | 推断来源 | 验证方式 |
|---|---|---|
| 密码 Base64 传输 | HashUtil.decodeBase64() | 抓包验证 |
| 登录错误 5 次锁定 | ACCOUNT_LOCKED_ERROR 存在 | 查询数据库 |

### 3.2 改密流

**完整路径**：
```
[用户] 输入旧密码 + 新密码
  → PUT /password (CoreLoginController)
  → checkPassword() 验证旧密码
  → changePass() 更新密码
```

**异常路径**：
| 异常场景 | 当前处理 | 期望处理 | 差距 |
|---|---|---|---|
| 旧密码错误 | REST_PASSWORD_ERROR | 同左 | ✅ |
| 新密码为空 | 未校验 | 应拒绝空密码 | 🔴 |
| 新旧密码相同 | 未校验 | 应拒绝相同密码 | 🟡 |

### 3.3 JIRA 数据采集流

**完整路径**：
```
[定时触发] CollectJobManager.execute()
  → JiraStoryJob 采集需求
  → JiraBugJob 采集 BUG
  → JiraBugReopenRecordJob 采集复现
  → 写入 Elasticsearch
```

**异常路径**：
| 异常场景 | 当前处理 | 期望处理 | 差距 |
|---|---|---|---|
| JIRA API 超时 | 未确认重试 | 应有重试+告警 | 🔴 |
| 采集频率过高 | 每分钟执行 | 可能触发限流 | 🟡 |
| 数据格式不匹配 | 静默跳过 | 应记录并告警 | 🔴 |

## 5. 事务边界分析

**@Transactional 使用情况**：

| Service | 方法 | 事务类型 | 传播行为 | 问题 |
|---|---|---|---|---|
| UserService | createUser() | REQUIRED | 默认 | ✅ |
| OrderService | batchCreate() | REQUIRED | 默认 | ⚠️ 大批量可能超时 |
| ExternalService | syncData() | REQUIRES_NEW | — | 🔴 未配置传播行为 |

**事务问题识别**：
| 问题 | 位置 | 严重度 | 建议 |
|---|---|---|---|
| @Transactional 在 public 方法上 | ServiceImpl:42 | 🟡 | 已正确 |
| 内部方法调用不生效 | save() 调用 innerSave() | 🔴 | 注入自身代理 |
| 异常被 catch 吞掉 | try { tx } catch {} | 🔴 | 重新抛出或编程式事务 |
